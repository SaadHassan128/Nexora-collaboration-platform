1. Core Philosophy

The architecture is designed around the principles of Obsidian Flux—a resilient, event-driven microservices ecosystem. It prioritizes decoupled deployment, strict type safety across boundaries, and sub-50ms latency for critical paths.

2. Data Flow & Event Bus

Communication between domains is strictly asynchronous, utilizing Apache Kafka for event sourcing. Synchronous calls are reserved solely for API Gateway to Edge-Service interactions.

event_publisher.ts 

import { KafkaClient } from '@nexora/events';
import type { ProjectEvent } from '@types/events';

// Initialize hardened client with retry logic
const client = new KafkaClient({
  clientId: 'core-api',
  brokers: process.env.KAFKA_BROKERS.split(','),
  ssl: true
});

export async function publishEvent(event: ProjectEvent): Promise<void> {
  try {
    await client.producer.send({
      topic: 'project.mutations',
      messages: [{ key: event.id, value: JSON.stringify(event) }]
    });
  } catch (error) {
    logger.error('Failed to publish event', { error, eventId: event.id });
    throw new EventPublishError(error);
  }
}

3. Technical Decisions

  • Database Sharding (Dec 2023) Transitioned from monolithic Postgres to Vitess for horizontal scaling of tenant data. Reduced P99 query latency by 40%.
  • Edge Compute Authentication (Jan 2024) Moved JWT validation to Cloudflare Workers to prevent unauthenticated requests from hitting the core infrastructure.
