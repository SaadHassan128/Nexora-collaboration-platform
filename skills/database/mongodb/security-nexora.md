## Purpose

Protect Nexora data using secure MongoDB practices.

---

# Sensitive Data

Protect:

* User credentials
* Private projects
* Team information
* Internal documentation

---

# Never Store

Never store:

* Plain passwords
* Raw tokens
* Secrets

---

# Access Control

Users should only access:

* Their own private data
* Projects they belong to
* Allowed resources

---

# Validation

Validate:

* Input data
* Schema shape
* User permissions

---

# Data Protection

Consider:

* Encryption
* Backups
* Secure connections

---

# Security Review

Before release:

Check:

* Authentication
* Authorization
* Data exposure
* Access rules

---

# Final Rule

Database security is part of product security.
