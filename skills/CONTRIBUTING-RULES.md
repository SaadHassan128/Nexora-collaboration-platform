# Nexora Skills Contribution Rules

## Purpose

This document defines how Nexora engineering skills should be used during development.

Skills are not isolated documents.

They represent the engineering standards that guide AI-assisted and human development.

---

# Before Starting Any Feature

Every feature should follow:

1. Understand product requirement
2. Review architecture impact
3. Design data changes
4. Implement feature
5. Test behavior
6. Review code quality

---

---

# Feature Development Rule

No feature should be implemented without considering:

- Architecture
- Database impact
- Security
- Testing

---

# AI Development Rules

When using AI tools:

AI should:

- Follow Nexora standards
- Respect existing architecture
- Avoid creating unnecessary complexity
- Ask before changing core decisions

---

# Review Requirement

Before merging:

Every change should pass:

- Architecture check
- Security check
- Testing check
- Code review

---

# Final Rule

Consistency is more important than speed.

Every contribution should make Nexora easier to grow.
