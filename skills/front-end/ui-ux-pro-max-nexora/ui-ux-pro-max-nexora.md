---

# Nexora Context

Nexora is a premium developer collaboration ecosystem.

The product connects:

- Developers
- Project Owners
- Teams

Through:

- Real projects
- Missions
- Collaboration
- Growth tracking
- Professional reputation


The design direction should communicate:

- Premium technology product
- Future of developer collaboration
- Trust
- Growth
- Innovation

---

# Design Personality

Nexora should feel:

- Premium
- Modern
- Professional
- Futuristic
- Human-centered
- Developer-focused


Avoid:

- Generic SaaS layouts
- Basic dashboard templates
- AI-generated looking interfaces
- Excessive gradients
- Random animations

---

# Visual Direction

Preferred styles:

Primary:

- Glassmorphism
- Spatial UI
- Motion-driven interfaces
- Bento layouts
- Dimensional layering
- Premium dark interfaces


Secondary:

- Minimalism
- Swiss modernism
- AI-native UI


Avoid:

- Overused purple AI gradients
- Heavy neon cyberpunk
- Gaming-style UI

---

# Nexora Color Philosophy

Colors should communicate:

Trust + Technology + Growth


Preferred:

- Dark premium backgrounds
- Controlled accent colors
- High contrast text
- Clear hierarchy


Do not choose colors randomly.

Every color should have a purpose:

Primary:

Brand identity

Accent:

Actions and important states

Success:

Growth and achievement

Warning:

Risk and attention

---

# Typography Rules

Typography should feel:

- Technical
- Professional
- Modern


Prioritize:

- Strong headings
- Clear readability
- Developer/product feel


Avoid:

- Decorative fonts
- Playful styles
- Low readability

---

# Component Design Rules

All UI components should be:

- Reusable
- Accessible
- Responsive
- Consistent


Examples:

Instead of:
ProjectCard1
ProjectCard2
ProjectCard3

Create:
ProjectCard

with configurable inputs

---

# Dashboard Rules

Nexora dashboards should focus on:

Information hierarchy.

Not:

Showing everything.


Prioritize:

- Progress
- Growth
- Contribution
- Team activity


Use:

- Cards
- Charts
- Timeline
- Interactive visualization

---

# 3D & Motion Guidelines

3D elements are allowed when they add meaning.

Good examples:

- Skill tree visualization
- Growth journey
- Product lifecycle
- Reputation progression


Avoid:

- Decorative 3D objects with no purpose
- Heavy animations affecting performance


Animations should be:

- Smooth
- Meaningful
- Short

---

# UX Principles

Every experience should answer:

"What does the user achieve here?"


Examples:

Developer:

Find opportunity → contribute → grow → showcase


Owner:

Create project → build team → deliver product


---

# Accessibility Requirements

Always consider:

- Keyboard navigation
- Clear focus states
- Proper contrast
- Responsive layouts


Support:

375px

768px

1024px

1440px

---

# Angular Implementation Rules

Follow:

Feature-based architecture.

Components should be:

- Small
- Focused
- Reusable


Large components should be split into:

- Child components
- Shared components
- Services


---

# Nexora UI Review Checklist

Before approving UI:

## Visual

- Does it look premium?
- Does it feel unique?
- Is spacing consistent?


## UX

- Is the user journey clear?
- Are actions obvious?


## Technical

- Is it reusable?
- Is it responsive?
- Is it maintainable?


---

# Final Rule

Nexora UI should not only look good.

It should communicate:

"Developers grow by building together."
